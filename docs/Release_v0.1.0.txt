DataScythe v0.1.0 - Release Notes
==============================

Release date: 2026-07-05

DataScythe reports kAppVersion 0.1.0 in src/core/version.h. The CLI --version
flag and GUI title bar reference this value.

Highlights
----------

- Initial promoted native release with YellowSphere-style repository layout.
- GNU shred-inspired pass scheduling from docs/reference/shred.c.
- Full-device wipe, quick zero-fill, volume shred, file/folder shred.
- NTFS alternate data stream shredding on Windows.
- Partition metadata wipe (first and last 1 MiB on block devices).
- Post-erase verification with sparse 512-byte samples.
- Erasure certificate export and GUI auto-save.
- Pre-flight checks with Administrator enforcement on Windows.
- Typed GUI confirmation (ERASE N, SHRED FILES, SHRED FOLDER) plus final Yes/No.
- CLI interactive YES confirmation (skippable with --yes and matching
  --confirm-target for drive-level modes).
- SSD hardware secure erase: NVMe Sanitize and ATA SECURITY ERASE on Windows.
- Detached PGP release signatures and SHA256SUMS manifest.
- Local Authenticode code-signing certificate for maintainer use.

PGP release signatures
----------------------

Detached ASCII-armored PGP signatures are included next to release artifacts as
.asc files. The release-signing public key is:

  docs/DataScythe_Release_Signing_2026_pubkey.asc

Release signing key:

  DataScythe Release Signing (2026) <release@datascythe.local>
  Fingerprint: 72E7 D550 77BA 8D7D 425B DC76 6133 2D0E 78FE F1CF

Verify the public key fingerprint before trusting signatures:

  gpg --show-keys docs/DataScythe_Release_Signing_2026_pubkey.asc

Verify the checksum manifest:

  gpg --verify docs/SHA256SUMS.asc docs/SHA256SUMS

Verify the Windows release archive:

  gpg --import docs/DataScythe_Release_Signing_2026_pubkey.asc
  gpg --verify releases/native/windows/DataScythe-0.1.0-win64.zip.asc releases/native/windows/DataScythe-0.1.0-win64.zip

Verify the Linux and macOS release archives:

  gpg --verify releases/native/linux/DataScythe-0.1.0-linux-amd64.tar.gz.asc releases/native/linux/DataScythe-0.1.0-linux-amd64.tar.gz
  gpg --verify releases/native/macos/DataScythe-0.1.0-macos.tar.gz.asc releases/native/macos/DataScythe-0.1.0-macos.tar.gz
  gpg --verify releases/native/macos/DataScythe-0.1.0-macos-universal.tar.gz.asc releases/native/macos/DataScythe-0.1.0-macos-universal.tar.gz

These PGP signatures provide release-integrity verification. They do not replace
Windows Authenticode signing for SmartScreen publisher trust.

Known limitations
-----------------

- Drive sizes may display as 0 without Administrator on Windows.
- NVMe SSDs may display as HDD in the drive list (cosmetic classification).
- Verification skips comparison when the final pass used random data.
- SSD crypto erase may not read as zeros during verification.
- Unit tests cover core logic only, not destructive hardware integration.
- Linux/macOS archives are CLI-only for v0.1.0.


Migration notes
---------------

See MIGRATION.md at the repository root for layout changes from earlier
prototypes. CLI binary in release zip is datascythe-cli.exe to avoid case
collision with DataScythe.exe on Windows.
